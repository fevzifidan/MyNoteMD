# MyNoteMD - Frontend Web Application

A modern, responsive, and highly modular frontend application built with React.js, and Tailwind CSS. This project strictly follows a **Feature-Based Architecture** to ensure scalable, maintainable, and clean code.

## ✨ Key Features

- **🤖 In-Browser Local AI:** Lightweight AI models can be run entirely on the client-side (browser) using WebGPU/WASM by providing proper .ai.env file. Your interaction with the AI ​​model remains entirely local.
- **🔐 Secure Authentication:** Opaque Token + Antiforgery Token-based authentication using React Context API with persistent sessions.
- **📝 Advanced Form Handling:** Built with `react-hook-form` and `yup` for complex validations (e.g., strong password policies requiring uppercase, lowercase, and non-alphanumeric characters).
- **🎨 Modern UI & Theming:** Utilizes **Shadcn UI** components. Fully supports Dark and Light modes with seamless transitions.
- **🧩 Feature-Based Architecture:** Domain-driven folder structure ensuring components, services, and logic are encapsulated by feature.
- **🌍 Key-Based Localization (i18n):** Built-in Language Selector for multi-language support.
- **📱 Fully Responsive:** Mobile-first approach ensuring a perfect layout across all device sizes.

## 🧠 Core Implementation Details

### Authentication Context and Secure Session Management (`AuthContext`)
In session management, the **Opaque Token** is stored in the HttpOnly cookie as a meaningless key representing the user session; this makes it impossible to steal the token through XSS attacks (via JavaScript). To protect against the risk of CSRF (Session Forgery) that can occur because cookies are automatically added by the browser to each request, we use the **Antiforgy Token** shield, which is generated by the server and which the client (React) must manually add as a header to each request.

### Form Validation
We strictly avoid inline validation logic. All forms use `yup` schemas separated into `validations/` folders. 
*Example: Our Registration form requires passwords to have at least 8 characters, one uppercase, one lowercase, and one special character.*

### Styling Rules
- **Tailwind CSS:** We exclusively use Tailwind CSS utility classes.
- **Shadcn Custom Variants:** We utilize the `cva` (class-variance-authority) pattern to manage UI states and specific design variants (e.g., `authInput`, `destructive` buttons).
- **Theme Variables:** Colors are defined in `globals.css` using CSS variables (e.g., `bg-background`, `text-muted-foreground`) to ensure Shadcn UI handles dark mode automatically.
